# 📡 Overview
Nexus-Flux exposes a **production-ready REST API** built with **FastAPI**.  
It provides a simple interface for submitting problems to the cognitive engine and retrieving the consensus solution.

All endpoints return JSON and follow standard HTTP conventions.

Base URL (default):
```
http://localhost:8000
```

To launch the API:
```bash
uvicorn src.api:app --reload
```

---

# 🔥 Main Endpoint

## **POST /solve**
Launches the Nexus-Flux cognitive process on a given problem statement.

### 📥 Request Body
```json
{
  "problem": "Your question or task here"
}
```

### Fields
| Field     | Type   | Required | Description |
|----------|--------|----------|-------------|
| problem  | string | yes      | The question, task, or prompt to solve |

---

### 🔁 Internal Flow
1. Initialize LLM service
2. Load state backend (memory or Redis)
3. Spawn cognitive agents
4. Run the Nexus-Flux engine (tick loop)
5. Compute consensus
6. Trigger PragmaticAgent action (background task)
7. Save execution log
8. Return consensus result

---

### 📤 Successful Response
```json
{
  "status": "success",
  "solution_text": "Final consensus explanation or answer.",
  "ticks": 7,
  "momentum": 0.0023,
  "consensus_vector": [0.12, -0.04, ...],
  "representative_node": {
    "text": "Synthesized key idea",
    "confidence": 0.87
  }
}
```

### Fields Explained
| Field | Description |
|-------|-------------|
| status | Always "success" unless an unexpected error occurs |
| solution_text | Textual synthesis of the consensus state |
| ticks | Number of cognitive cycles executed |
| momentum | Final momentum value after convergence |
| consensus_vector | Embedding of the converged cognitive state |
| representative_node | The most central node in the final state |

---

### ⚠️ Error Response Example
```json
{
  "detail": "Invalid problem statement. Expected a non-empty string."
}
```

HTTP Codes:
- **422** → Validation error
- **500** → Internal engine failure

```python
class SolveResponse(BaseModel):
```

# 🔧 Pragmatic Agent Execution
After generating the consensus, Nexus-Flux automatically triggers:
```
PragmaticAgent.execute()
```
This generates:
- a Markdown file (`solution.md`)
- summarization of problem + final answer

Execution happens asynchronously in background tasks.

# 🧭 Example: Full API Call (cURL)
```bash
curl -X POST http://localhost:8000/solve \
  -H "Content-Type: application/json" \
  -d '{"problem": "Explain general relativity in simple terms"}'
```

# 🧭 Example: Full API Call (Python)
```python

    "http://localhost:8000/solve",
    json={"problem": "Invent a startup idea for West Africa"}
)
print(resp.json())
```

# 🛑 Notes
- API errors are logged in structured JSON format.
- Long problems may require increasing timeout values.
- PragmaticAgent can be disabled via `.env`.
