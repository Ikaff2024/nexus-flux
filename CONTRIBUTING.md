# Contributing to Nexus-Flux

Thank you for your interest in contributing to **Nexus-Flux** — a next-generation cognitive engine created by **YEO Kaffanan Issa** and supported by **IKAFFANAN LTD (UK)**.

Your contributions help push the boundaries of AI research and engineering.

---

## 🚀 How to Contribute
There are several ways you can contribute:

### **1. Report Issues**
If you encounter bugs, inconsistencies, or unexpected behaviour:
- Open a GitHub Issue
- Provide clear steps to reproduce
- Include logs, stack traces, or screenshots when possible

### **2. Suggest Enhancements**
Have an idea to improve the engine?
- Submit a Feature Request
- Explain the use-case and expected behaviour

### **3. Submit Pull Requests**
We welcome high-quality PRs.

Before submitting:
- Fork the repository
- Create a dedicated feature branch: `feature/my-improvement`
- Follow our coding style and architecture principles
- Add tests for new behaviour when possible
- Ensure all tests pass (`pytest`)
- Ensure the code is type-hinted and documented

PRs that follow the guidelines are reviewed faster.

---

## 📐 Code Style Guidelines
To maintain consistency:

- Use **Python 3.10+**
- Use **type hints** for all functions and classes
- Use **async/await** for all agent and state operations
- Keep functions small and focused
- Use **snake_case** for variables and functions
- Use **PascalCase** for classes
- Follow existing folder structure
- Write meaningful commit messages

---

## 🧪 Testing
All new features must include appropriate tests.

Run tests locally:
```bash
pytest
```

PRs without tests may be delayed.

---

## 🔒 Security
If you discover a security vulnerability, **do NOT open a public issue**.

Instead, contact:
- **security@ikaffanan.com** (official security contact)

We will respond as quickly as possible.
